Лабораторная работа №5 — что исправлено

1. Исправлена регистрация пользователя:
- форма теперь отправляется методом POST;
- добавлен csrf_token;
- используются встроенные механизмы Django UserCreationForm;
- проверяется уникальность email;
- проверяется совпадение паролей;
- после успешной регистрации пользователь создаётся в БД и автоматически входит в систему.

2. Проверка шифрования пароля:
- регистрация выполняется через UserCreationForm/save(), поэтому пароль не хранится как обычный текст;
- в tests.py добавлен тест, который проверяет user.check_password(...);
- в базе данных в поле auth_user.password будет храниться хэш вида pbkdf2_...

3. Дополнительные модели и расширение пользователя:
- добавлена модель UserProfile через OneToOneField к пользователю;
- это соответствует рекомендации из статьи Habr по безопасному расширению пользователя;
- создана миграция 0003_userprofile_alter_likedislike_user.py.

4. Прочие исправления:
- исправлены __str__ методы моделей;
- улучшен like/dislike: теперь используется request.user и требуется авторизация;
- секреты Telegram вынесены в переменные окружения;
- модели зарегистрированы в admin.py.

5. Проверка кода:
- Python-файлы проверены на синтаксис через py_compile.

Важно:
Если проект запускать заново в чистой среде, выполнить:
python manage.py migrate
python manage.py runserver
