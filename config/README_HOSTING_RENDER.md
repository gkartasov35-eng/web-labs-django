# Развёртывание проекта на хостинг Render

Проект подготовлен для деплоя через Docker. Главный Django-проект находится в папке `config`.

## Вариант 1. Через Blueprint

1. Загрузить проект в GitHub.
2. Если в репозиторий загружается вся папка архива, оставить `render.yaml` в корне архива.
3. Если в репозиторий загружается только содержимое папки `config`, использовать `render.yaml` из этой папки.
4. На Render выбрать **New → Blueprint** и указать GitHub-репозиторий.
5. Render создаст Docker Web Service `web-labs-django`.

## Вариант 2. Вручную через Web Service

1. Render → **New → Web Service**.
2. Подключить GitHub-репозиторий.
3. Если репозиторий содержит всю папку архива, в поле **Root Directory** указать `config`.
4. Environment выбрать **Docker**.
5. Start Command можно не указывать, потому что команда запуска уже прописана в `Dockerfile`.
6. Добавить переменные окружения:

```text
DEBUG=0
SECRET_KEY=<сгенерированный секретный ключ>
ALLOWED_HOSTS=.onrender.com,localhost,127.0.0.1
CSRF_TRUSTED_ORIGINS=https://*.onrender.com
PORT=8000
```

После успешного деплоя сайт будет доступен по адресу вида:

```text
https://web-labs-django.onrender.com
```

Точный адрес нужно взять из Render Dashboard после деплоя.

## Команды, которые выполняются внутри контейнера

```bash
python manage.py collectstatic --noinput
python manage.py migrate
gunicorn config.wsgi:application --bind 0.0.0.0:${PORT:-8000}
```

## Проверка после деплоя

1. Открыть главную страницу `/`.
2. Перейти на `/auth/` и зарегистрировать нового пользователя.
3. Проверить, что после регистрации пользователь авторизован.
4. На главной странице нажать like/dislike и увидеть обновление без перезагрузки.
5. Открыть `/feedback/`, отправить сообщение и увидеть JSON/AJAX-ответ.

## Важное замечание про базу данных

По умолчанию проект использует SQLite, потому что это требование учебной лабораторной и самый простой вариант для проверки.
Для постоянного хранения данных на настоящем хостинге лучше подключить PostgreSQL и задать переменную `DATABASE_URL`.
Настройки уже подготовлены: если `DATABASE_URL` есть, Django автоматически использует PostgreSQL.
